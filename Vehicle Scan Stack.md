Vehicle Scan Stack

Gridee_Android/android-app/app/src/main/java/com/gridee/parking/ui/operator/OperatorDashboardActivity.kt – exposes the “Scan Vehicle” button, launches the scanner with the correct intent extras, and reacts to the returned plate to trigger operator check‑in/out via the view model.
Gridee_Android/android-app/app/src/main/java/com/gridee/parking/ui/operator/OperatorViewModel.kt – wraps the repository calls for operator check‑in / check‑out using vehicle numbers and publishes success/error state back to the dashboard UI.
Gridee_Android/android-app/app/src/main/java/com/gridee/parking/ui/qr/QrScannerActivity.kt – dual‑mode scanner; in operator mode it runs CameraX + ML Kit text recognition, handles the overlay, feedback, 7‑second timeout, and emits the detected plate.
Gridee_Android/android-app/app/src/main/res/layout/activity_qr_scanner.xml – layout hosting the ZXing view plus the CameraX preview, scan frame, animated line, and status chip shown during vehicle scans.
Gridee_Android/android-app/app/src/main/java/com/gridee/parking/data/repository/BookingRepository.kt – provides operatorCheckIn / operatorCheckOut methods that OperatorViewModel calls once a vehicle number is scanned.
Gridee_Android/android-app/app/src/main/java/com/gridee/parking/data/api/ApiService.kt – defines the Retrofit endpoints (/api/bookings/checkin and /api/bookings/checkout) used for operator check‑ins via vehicle number.
Gridee_Android/android-app/app/src/main/java/com/gridee/parking/data/model/CheckInMode.kt & CheckInRequest.kt – model enums/data classes that mark the operator requests as VEHICLE_NUMBER mode with the captured plate text.
Gridee_Android/android-app/app/src/main/res/values/strings.xml plus the overlay drawables (scanner_overlay.xml, vehicle_scan_line.xml, vehicle_status_background.xml) – provide the copy and visuals for the on‑scanner feedback and dialogs.
Those files together cover the UI surface, scanning pipeline, and backend calls for operator vehicle check‑ins.
